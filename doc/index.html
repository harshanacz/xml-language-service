XML Language Service

A pure-TypeScript library that provides full XML language intelligence — the same features that power IDE extensions — without coupling to any editor. Drop it into a VS Code extension, a web editor, or a CLI tool.

$ npm install xml-language-service
Fault-tolerant parsing XSD validation (WASM) LSP-shaped types Zero editor deps

Auto Completion

Context-aware suggestions for elements, attributes, and closing tags.

Hover Info

Rich Markdown hover content showing element structure and attributes.

Document Symbols

Full hierarchical outline of the XML document, ready for an editor tree view.

Rename

Rename both opening and closing tags atomically with a single call.

Go to Definition

Jump between matching open/close tag pairs instantly.

XSD Validation

Full XSD schema validation via libxml2 compiled to WebAssembly.

Installation

Install via npm, yarn, or pnpm. The package ships as ESM with full TypeScript typings.

bash
npm install xml-language-service
# or
yarn add xml-language-service
# or
pnpm add xml-language-service

Requires Node.js 16+. The libxml2-wasm peer dependency is bundled automatically.

tsconfig requirements

The package is pure ESM. Make sure your tsconfig.json targets a module format that supports it:

json — tsconfig.json
{
  "compilerOptions": {
    "module": "ESNext",
    "moduleResolution": "bundler"
  }
}

Quick Start

Create one service instance at startup, keep it alive, and call dispose when you're done.

typescript
import { getLanguageService } from "xml-language-service";

// Create once — owns WASM memory internally
const ls = getLanguageService();

const xmlText = `
<catalog>
  <book id="bk101">
    <author>Gambardella, Matthew</author>
    <title>XML Developer's Guide</title>
  </book>
</catalog>
`.trim();

// Parse
const doc = ls.parseXMLDocument("file:///catalog.xml", xmlText);

// Completion at cursor
const completions = ls.doComplete(doc, { line: 1, character: 3 });
console.log(completions.items.map(c => c.label));
// → ["catalog", "book", "author", "title"]

// Hover
const hover = ls.doHover(doc, { line: 1, character: 3 });
console.log(hover?.contents);
// → "### <book>\n**Attributes:** 1 — `id`\n**Children:** 2"

// Document symbols (outline)
const symbols = ls.findDocumentSymbols(doc);

// Format
const edits = ls.format(doc, { tabSize: 2, insertSpaces: true });
// edits[0].newText → properly indented XML string

// Always dispose when done to free WASM memory
ls.dispose();

getLanguageService()

The sole public entry point. Returns a stateful service object that owns one internal SchemaProvider (XSD registry + WASM memory). Create one instance per application lifetime.

getLanguageService () → LanguageService sync

Creates and returns a new LanguageService object. No arguments required.

Returns

A LanguageService object with all methods described in this reference, plus dispose().

Always call ls.dispose() when you no longer need the service. This frees the libxml2-wasm WASM memory held by the schema registry.

parseXMLDocument()

Parses an XML string into an XMLDocument AST. Fault-tolerant — never throws, even on broken input.

parseXMLDocument (uri, text) → XMLDocument sync
ParameterTypeDescription
uristringDocument identifier, e.g. "file:///path/to/file.xml". Used as the uri field on the returned document and in result objects.
textstringRaw XML source text.
Returns

An XMLDocument — a normalized AST with findNodeAt(offset) and traverse(callback) helpers. Pass this object to any other service method.

typescript
const doc = ls.parseXMLDocument("file:///my.xml", "<root><child /></root>");
console.log(doc.children[0].name);   // "root"
console.log(doc.children[0].children[0].isSelfClosing); // true

doComplete()

Returns context-aware completion items at a given cursor position.

doComplete (document, position) → CompletionList sync
ParameterTypeDescription
documentXMLDocumentParsed document from parseXMLDocument().
positionPositionZero-based { line, character } cursor position.
Completion contexts
  • After </ — suggests the matching close tag name. kind: "closeTag"
  • After < or partial tag name — suggests all element names in the doc. kind: "element", insertText tagName>$0</tagName>
  • In attribute position (after tag name + space) — suggests xml:lang, xml:space, xmlns. kind: "attribute"
  • Otherwise — empty list.
typescript
const doc = ls.parseXMLDocument("file:///x.xml", "<root><child/></root>");

// Element completion
const list = ls.doComplete(doc, { line: 0, character: 1 });
list.items.forEach(i => console.log(i.label, i.insertText));
// "root"  →  "root>$0</root>"
// "child" →  "child>$0</child>"

// Attribute completion
const attrs = ls.doComplete(doc, { line: 0, character: 6 });
attrs.items.map(i => i.label);
// ["xml:lang", "xml:space", "xmlns"]

doHover()

Returns Markdown hover content for the XML node under the cursor.

doHover (document, position) → HoverResult | null sync
ParameterTypeDescription
documentXMLDocumentParsed document.
positionPositionCursor position.
Returns

HoverResult for element and comment nodes, null for the document root and plain text nodes.

typescript
const doc = ls.parseXMLDocument("file:///x.xml",
  '<root id="1"><child/></root>');

const h = ls.doHover(doc, { line: 0, character: 1 });
console.log(h?.contents);
// "### <root>\n**Attributes:** 1 — `id`\n**Children:** 1"

findDocumentSymbols()

Builds a hierarchical symbol tree from all element nodes — ideal for populating an editor's outline panel.

findDocumentSymbols (document) → DocumentSymbol[] sync
ParameterTypeDescription
documentXMLDocumentParsed document.
Returns

An array of DocumentSymbol objects. Each symbol has a range (full element) and a selectionRange (just the <tagName part). Nested elements appear as children.

typescript
const symbols = ls.findDocumentSymbols(doc);
// [
//   {
//     name: "catalog",
//     kind: "element",
//     range: { start: {line:0, character:0}, end: {line:5, character:11} },
//     selectionRange: { ... },
//     children: [
//       { name: "book", kind: "element", children: [...], ... }
//     ]
//   }
// ]

getFoldingRanges()

Returns all collapsible regions for multi-line, non-self-closing elements.

getFoldingRanges (document) → FoldingRange[] sync
ParameterTypeDescription
documentXMLDocumentParsed document.
Returns

Flat array of FoldingRange. Each has startLine, endLine, and kind: "region".

typescript
const folds = ls.getFoldingRanges(doc);
// [{ startLine: 0, endLine: 5, kind: "region" }, ...]

format()

Re-indents the document and returns a single TextEdit replacing the entire source. Does not mutate the AST.

format (document, options?) → TextEdit[] sync
ParameterTypeDescription
documentXMLDocumentParsed document.
optionsFormatterOptions?Optional formatting settings (see below).

FormatterOptions

PropertyTypeDefaultDescription
tabSizenumber2Spaces per indent level.
insertSpacesbooleantrueUse spaces instead of tabs.
maxLineLengthnumber?—Accepted but not currently enforced.
Returns

A single-element TextEdit[] whose newText is the fully formatted document. Apply by replacing document.text from startOffset to endOffset. Returns [] for empty documents.

typescript
const doc = ls.parseXMLDocument("file:///x.xml",
  "<root><child><leaf/></child></root>");

const [edit] = ls.format(doc, { tabSize: 2, insertSpaces: true });
console.log(edit.newText);
// <root>
//   <child>
//     <leaf/>
//   </child>
// </root>

doRename()

Renames both the opening and closing tags of the element under the cursor atomically.

doRename (document, position, newName) → TextEdit[] | null sync
ParameterTypeDescription
documentXMLDocumentParsed document.
positionPositionCursor on an element tag.
newNamestringThe replacement tag name.
Returns

TextEdit[] — two edits for regular elements (open + close), one for self-closing. Returns null if newName is empty, contains <, >, or spaces, or if the cursor isn't on a named element.

typescript
const doc = ls.parseXMLDocument("file:///x.xml", "<root><child/></root>");
const edits = ls.doRename(doc, { line: 0, character: 1 }, "container");
// edits[0] → rename open tag  <root>   → <container>
// edits[1] → rename close tag </root>  → </container>

doDefinition()

Navigates between a matching open/close tag pair — "go to definition" for XML tags.

doDefinition (document, position) → DefinitionResult | null sync
ParameterTypeDescription
documentXMLDocumentParsed document.
positionPositionCursor on an element opening or closing tag.
Returns

DefinitionResult pointing to the opposite tag. null for self-closing elements.

typescript
// Cursor on open tag → result points to close tag
const def = ls.doDefinition(doc, { line: 0, character: 1 });
// { uri: "file:///x.xml", range: { start: {...}, end: {...} } }

findReferences()

Finds matching elements by tag name, and when available, narrows matches using identifying attributes such as name, id, key, or ref.

findReferences (document, position) → ReferenceResult[] sync
ParameterTypeDescription
documentXMLDocumentParsed document.
positionPositionCursor on a named element.
Returns

Array of ReferenceResult — one entry per matching element. Matching uses tag name, and also identifying attribute name/value pairs when present on the cursor element. Returns [] if not on a named element.

typescript
const refs = ls.findReferences(doc, { line: 1, character: 3 });
// All <book> elements in the document:
// [{ uri: "file:///catalog.xml", range: {...} }, ...]

Schema Validation

XSD validation is powered by libxml2 compiled to WebAssembly. Register XSD schemas by URI, then validate any parsed document against a registered schema.

registerSchema (info: SchemaInfo) → Promise<void> async

Compiles an XSD string and registers it under a URI key. Replaces any existing schema at the same URI (old WASM memory is freed automatically).

ParameterTypeDescription
info.uristringUnique identifier for the schema (used later to call validate()).
info.xsdTextstringRaw XSD source text.
validate (schemaUri, document) → Promise<Diagnostic[]> async

Validates a parsed document against a registered XSD schema. Never throws.

ParameterTypeDescription
schemaUristringURI of a previously registered schema.
documentXMLDocumentDocument to validate.
Returns

Diagnostic[] — empty array means valid. If schemaUri is not registered, returns a single severity: "warning" diagnostic.

hasSchema (uri: string) → boolean sync

Returns true if a schema is registered under uri.

dispose () → void sync

Frees all WASM memory held by the schema registry. Call this when the service is no longer needed.

Full XSD validation example

typescript
import { getLanguageService } from "xml-language-service";

const ls = getLanguageService();

const xsd = `<?xml version="1.0"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="root">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="child" type="xs:string" minOccurs="0"/>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>`;

// Register
await ls.registerSchema({ uri: "file:///schema.xsd", xsdText: xsd });

// Valid document
const valid = ls.parseXMLDocument("file:///ok.xml",
  '<?xml version="1.0"?><root><child>hello</child></root>');
const okDiags = await ls.validate("file:///schema.xsd", valid);
console.log(okDiags); // []

// Invalid document
const bad = ls.parseXMLDocument("file:///bad.xml",
  '<?xml version="1.0"?><root><unknown/></root>');
const errors = await ls.validate("file:///schema.xsd", bad);
// [{ message: "...", severity: "error", source: "xsd", range: {...} }]

ls.dispose();

AST Node Types

These interfaces describe the document tree returned by parseXMLDocument().

interface XMLDocument 
interface XMLDocument extends XMLNode {
  type: "root";
  uri: string;         // Document identifier
  text: string;        // Raw source text
  findNodeAt(offset: number): XMLNode;   // Deepest node at char offset
  traverse(callback: (node: XMLNode) => void): void; // Pre-order DFS
}
interface XMLNode 
interface XMLNode {
  type: "element" | "text" | "comment" | "root";
  name: string | undefined;
  attributes: XMLAttribute[];
  children: XMLNode[];
  startOffset: number;
  endOffset: number;
  parent: XMLNode | undefined;
  isSelfClosing: boolean;
}
interface XMLAttribute 
interface XMLAttribute {
  name: string;
  value: string | undefined;    // Quote-stripped value
  nameStart: number;            // Char offset of name start
  nameEnd: number;              // Char offset of name end
  valueStart: number | undefined;  // Offset after opening quote
  valueEnd: number | undefined;    // Offset before closing quote
}
interface Position / Range 
interface Position {
  line: number;       // 0-based
  character: number;  // 0-based
}

interface Range {
  start: Position;
  end: Position;
}

Using XsdValidatorService directly

For low-level control, XsdValidatorService is also exported:

import { XsdValidatorService } from "xml-language-service";

const validator = await XsdValidatorService.create(xsdText);
const diagnostics = await validator.validate(xmlText); // never throws
validator.dispose();

LSP Integration

All result types are modeled after the Language Server Protocol. Bridging to an LSP server is straightforward.

✅

Position, Range, DocumentSymbol, FoldingRange, Diagnostic, DefinitionResult all mirror LSP types exactly. Only TextEdit uses character offsets instead of Range — convert with offsetToPosition() if needed.

VS Code extension sketch

typescript — extension.ts
import * as vscode from "vscode";
import { getLanguageService } from "xml-language-service";

const ls = getLanguageService();

export function activate(ctx: vscode.ExtensionContext) {
  ctx.subscriptions.push(
    vscode.languages.registerCompletionItemProvider("xml", {
      provideCompletionItems(document, position) {
        const doc = ls.parseXMLDocument(document.uri.toString(), document.getText());
        const list = ls.doComplete(doc, position);
        return list.items.map(item => {
          const ci = new vscode.CompletionItem(item.label);
          ci.insertText = new vscode.SnippetString(item.insertText);
          return ci;
        });
      }
    }, "<", "/"),

    vscode.languages.registerHoverProvider("xml", {
      provideHover(document, position) {
        const doc = ls.parseXMLDocument(document.uri.toString(), document.getText());
        const hover = ls.doHover(doc, position);
        if (!hover) return;
        return new vscode.Hover(hover.contents);
      }
    })
  );
}

export function deactivate() { ls.dispose(); }

Architecture

A high-level view of how data flows through the library.

TextDocument (uri + raw string) │ ▼ parseXMLDocument() → @xml-tools/parser (Chevrotain CST) │ ▼ XMLDocumentImpl (normalized node tree) │ ├─── doComplete() completion items ├─── doHover() markdown hover ├─── findDocumentSymbols() hierarchical outline ├─── getFoldingRanges() collapsible regions ├─── format() indented text edits ├─── doRename() tag pair text edits ├─── doDefinition() matching tag range ├─── findReferences() same-name elements │ └─── SchemaProvider (URI → XsdValidatorService) │ └─── XsdValidatorService → libxml2-wasm

Key design decisions

  • Immutable documents — services read XMLDocument but never mutate it; format() works on the text string only.
  • Fault tolerance — the parser (via @xml-tools/parser) and the validator (try/catch wrapping) handle broken input without throwing.
  • LSP-compatible shapes — Position, Range, DocumentSymbol, FoldingRange, Diagnostic all mirror LSP types, making it straightforward to adapt for LSP servers.
  • WASM memory management — XsdValidatorService and SchemaProvider both expose dispose() which must be called to free libxml2-wasm memory.
  • Schema as a registry — SchemaProvider is a URI-keyed Map of compiled validators. Registering the same URI twice disposes the old validator before replacing it.
